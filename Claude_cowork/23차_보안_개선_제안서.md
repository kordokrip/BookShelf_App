## 1. 보안 이슈 (SEC)

### 🔴 SEC-01: 사용자 프로필 비인증 조회 (데이터 노출)

**파일**: `worker/routes/users.ts` — `GET /api/users/:id`

**현황**: `authMiddleware` 없이 모든 사용자의 이름, 이메일, 아바타, 역할, 가입일을 조회할 수 있다. `password_hash`는 `safeUser()`로 제외되지만 나머지 개인 정보가 노출된다.

```typescript
// 현재 코드 — 인증 없음
usersRouter.get('/:id', async (c) => {
  const id = c.req.param('id');
  const user = await c.env.DB.prepare(
    'SELECT * FROM users WHERE id = ?',
  ).bind(id).first<DbUser>();
  return c.json({ data: safeUser(user) });
});
```

**위험**: 이메일 수집, 계정 열거, 프라이버시 침해

**제안**:
```typescript
usersRouter.get('/:id', authMiddleware, async (c) => {
  const requesterId = c.get('userId');
  const id = c.req.param('id');
  const user = await c.env.DB.prepare(
    'SELECT * FROM users WHERE id = ?',
  ).bind(id).first<DbUser>();
  if (!user) throw new HTTPException(404, { message: '사용자를 찾을 수 없습니다.' });

  // 자신의 프로필은 전체 조회, 타인은 공개 필드만
  if (requesterId === id) {
    return c.json({ data: safeUser(user) });
  }
  return c.json({ data: { id: user.id, name: user.name, avatar_url: user.avatar_url, profile_emoji: user.profile_emoji } });
});
```

### 🔴 SEC-02: Auth Refresh 엔드포인트 Rate Limiting 미적용

**파일**: `worker/routes/auth.ts` — `POST /api/auth/refresh`

**현황**: 로그인(5/min)과 회원가입(3/min)에는 Rate Limiting이 있지만, Refresh Token 엔드포인트에는 없다.

```typescript

  const { refreshToken } = await c.req.json<{ refreshToken: string }>();
  // ...
```

**위험**: Refresh Token 무차별 대입 공격, 토큰 열거, DoS

```typescript
authRouter.post(
  '/refresh',
  rateLimit({ limit: 10, windowMs: 60_000, keyPrefix: 'refresh' }),
  async (c) => { /* ... */ }
);
```

### 🟡 SEC-04: 페이지네이션 limit 파라미터 최댓값 미검증

**파일**: `worker/routes/books.ts` GET `/api/books`, `worker/routes/share.ts` GET `/inbox`, `/sent`

**현황**: `books.ts`에서 `limit`을 `parseInt(c.req.query('limit') ?? '100')`로 파싱하지만 최댓값 제한이 없다. `?limit=999999`으로 전체 데이터셋을 요청 가능하다. `share.ts`의 inbox/sent는 `LIMIT 50`이 하드코딩되어 페이지네이션이 불가능하다.

```typescript
const limit = Math.min(Math.max(1, parseInt(c.req.query('limit') ?? '100')), 500);
const offset = Math.max(0, parseInt(c.req.query('offset') ?? '0'));

// share.ts — inbox/sent에도 동적 페이지네이션 적용
```

### 🟡 SEC-06: Refresh Token을 localStorage에 저장

**파일**: `src/lib/api.ts`, `src/stores/authStore.ts`

**현황**: Access Token과 Refresh Token이 모두 `localStorage`에 저장된다. XSS 취약점이 하나라도 존재하면 Refresh Token(30일 유효)까지 탈취 가능하다.

```typescript
localStorage.setItem(TOKEN_KEY, token);
localStorage.setItem(REFRESH_TOKEN_KEY, refreshToken);
```

**위험**: XSS → 장기 유효 토큰 탈취 → 30일간 계정 접근

**제안**:
- Refresh Token은 **HTTP-only, Secure, SameSite=Strict 쿠키**로 이전
- Access Token(1시간)만 localStorage 유지
- `POST /api/auth/refresh`에서 쿠키로 자동 전달

```typescript
// 백엔드 — refresh 시 쿠키 설정
c.header('Set-Cookie',
  `refreshToken=${token}; HttpOnly; Secure; SameSite=Strict; Path=/api/auth; Max-Age=${60*60*24*30}`
);

// 프론트엔드 — fetch에 credentials 포함
fetch('/api/auth/refresh', { method: 'POST', credentials: 'include' });
```

### 🟡 SEC-07: JWT Access Token 유효기간 과다 (24시간)

**파일**: `worker/auth.ts` — `createToken()`

```typescript
  { ...payload, iat: now, exp: now + 3600 },  // 1시간
```

## 2. 성능 최적화 (PERF)

### 🔴 PERF-01: 채팅 폴링 → WebSocket 전환 (Durable Objects)

**파일**: `src/hooks/useGroups.ts`, `worker/routes/groups.ts`

**현황**: `useGroupMessages`가 10초 폴링으로 D1 쿼리를 반복한다. 22차에서 `refetchIntervalInBackground: false`로 최적화했지만, 근본적으로 폴링은 비효율적이다.

**영향**: 50명 그룹 5개 활성 시 분당 ~1,500회 D1 쿼리

**제안**: Cloudflare Durable Objects + WebSocket으로 전환.
```typescript
// worker/durable-objects/ChatRoom.ts
export class ChatRoom implements DurableObject {
  sessions: Map<WebSocket, { userId: string }> = new Map();

  async fetch(request: Request) {
    if (request.headers.get('Upgrade') !== 'websocket') {
      return new Response('Expected WebSocket', { status: 400 });
    }
    const pair = new WebSocketPair();
    this.sessions.set(pair[1], { userId: '' });
    pair[1].accept();
    pair[1].addEventListener('message', (msg) => {
      // D1에 영속화 + 모든 세션에 브로드캐스트
      for (const [ws] of this.sessions) {
        if (ws !== pair[1] && ws.readyState === WebSocket.OPEN) {
          ws.send(msg.data);
        }
      }
    });
    return new Response(null, { status: 101, webSocket: pair[0] });
  }
}
```

### 🔴 ARCH-02: 에러 응답 형식 불일치

**파일**: 모든 worker/routes/*.ts

**현황**: 에러 응답이 4가지 이상의 형식으로 혼재한다:
```typescript
return c.json({ error: 'message' }, 400);              // 형식 1: { error: string }
```

```typescript
// worker/types.ts
interface ApiError {

// worker/index.ts — onError 표준화
app.onError((err, c) => {
  if (err instanceof HTTPException) {

  console.error('[unhandled]', err);
  return c.json({ error: '서버 오류가 발생했습니다.', code: 'INTERNAL' }, 500);
```

### 🟡 ARCH-03: 구조화된 로깅 미도입

**현황**: `console.error/warn/log`를 사용하지만 구조화 없이 문자열 출력만 한다. 요청 추적(request ID)이 없어 프로덕션 디버깅이 어렵다.

**제안**:
```typescript
// worker/index.ts — 요청 추적 미들웨어
app.use('*', async (c, next) => {
  const requestId = crypto.randomUUID().slice(0, 8);
  c.set('requestId', requestId);
  const start = Date.now();
  try {
    await next();
  } finally {
    console.log(JSON.stringify({
      rid: requestId,
      method: c.req.method,
      path: c.req.path,
      status: c.res.status,
      ms: Date.now() - start,
    }));
  }
});
```

### 🟡 ARCH-04: JWT 검증 시 타입 강제 캐스팅

```typescript
if (

const payload = decoded as JwtPayload;
```

## 4. 인프라 & 운영 (OPS)

### 🔴 OPS-01: 헬스체크에 DB 연결 확인 누락

**파일**: `worker/index.ts` — `GET /api/health`

**현황**: 헬스체크가 단순히 `{ status: 'ok' }`만 반환한다. D1 데이터베이스가 다운되어도 200 OK를 반환하므로 장애 감지가 불가능하다.

```typescript
app.get('/api/health', (c) =>
  c.json({ status: 'ok', env: c.env.ENVIRONMENT, timestamp: new Date().toISOString() }),
);
```

**제안**:
```typescript
app.get('/api/health', async (c) => {
  const checks: Record<string, 'ok' | 'fail'> = {};
  try {
    await c.env.DB.prepare('SELECT 1').first();
    checks.database = 'ok';
  } catch { checks.database = 'fail'; }

  try {
    await c.env.KV.get('_healthcheck');
    checks.kv = 'ok';
  } catch { checks.kv = 'fail'; }

  const healthy = Object.values(checks).every((v) => v === 'ok');
  return c.json(
    { status: healthy ? 'healthy' : 'degraded', checks, timestamp: new Date().toISOString() },
    healthy ? 200 : 503,
  );
});
```
