## 2. 백엔드 리팩토링

### ✅ ~~🟡 MEDIUM — `cover_image` 필드 이중 의미~~ **[RESOLVED — 19차]**

**해결**: `resolveBookCoverUrl()` 함수 도입. R2 저장 시 `r2://` prefix 적용, 목록/상세 API에서 `r2://` prefix를 감지하여 `/api/books/:id/cover` URL로 자동 변환. 레거시 raw key도 하위 호환 유지.

---

### ✅ ~~🟡 MEDIUM — reading_sessions 중복 방지 없음~~ **[RESOLVED — 18차]**

**해결**: `POST /api/sessions` 핸들러에 10초 이내 동일 세션(book_id + session_date + pages_read) 중복 체크 로직 추가. 중복 시 기존 세션 반환 (200 OK).

---

### ✅ ~~🟢 LOW — is_overdue 필드 수동 관리~~ **[RESOLVED — 18차]**

**해결**: `GET /api/books`, `GET /api/books/:id` 쿼리에서 `is_overdue`를 동적 계산:
```sql
CASE WHEN goal_date IS NOT NULL AND goal_date < date('now') AND status = 'reading' THEN 1 ELSE 0 END AS is_overdue
```
