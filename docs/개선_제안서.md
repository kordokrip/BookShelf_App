## 2. 백엔드 리팩토링

### 🟡 MEDIUM — `cover_image` 필드 이중 의미

**현재 상태**: `books.cover_image`에 R2 키(`covers/userId/bookId.jpg`)와 외부 URL(`https://...`)이 혼재. `GET /api/books/:id/cover`에서 `startsWith('http')` 분기로 처리 중. `normalizeCoverUrl()`이 CDN URL을 프록시 URL로 변환.

> ⚠️ 현재 동작에 문제 없으나, 향후 R2 키에 `r2://` prefix 규약 적용 시 더 명확해짐. **후순위 개선 대상.**

---

### ✅ ~~🟡 MEDIUM — reading_sessions 중복 방지 없음~~ **[RESOLVED — 18차]**

**해결**: `POST /api/sessions` 핸들러에 10초 이내 동일 세션(book_id + session_date + pages_read) 중복 체크 로직 추가. 중복 시 기존 세션 반환 (200 OK).

---

### ✅ ~~🟢 LOW — is_overdue 필드 수동 관리~~ **[RESOLVED — 18차]**

**해결**: `GET /api/books`, `GET /api/books/:id` 쿼리에서 `is_overdue`를 동적 계산:
```sql
CASE WHEN goal_date IS NOT NULL AND goal_date < date('now') AND status = 'reading' THEN 1 ELSE 0 END AS is_overdue
```

## 5. 성능 최적화

### 🟡 MEDIUM — 앱 최초 진입 시 동시 요청 폭발

**현재 상태**: BottomNavBar(2개) + 페이지 훅(2~4개) → 앱 첫 진입 시 6~8개 동시 요청.

**제안**:
1. `suspense: true` + React Suspense로 워터폴 제어
2. 또는 `/api/initial-data` 일괄 로드 엔드포인트 도입 (서버에서 D1 batch)

> ⚠️ 현재 TanStack Query 캐싱(staleTime 60s)으로 실제 체감 영향 미미. **후순위.**

---

### 🟢 LOW — PWA precache 39 entries (1944.57 KiB)

**현재 상태**: 빌드 시 precache 크기가 ~2MB. 느린 네트워크 환경에서 Service Worker 설치 시간 증가.

**제안**: 자주 변경되지 않는 정적 자산만 precache, 나머지는 runtime caching으로 전환.
